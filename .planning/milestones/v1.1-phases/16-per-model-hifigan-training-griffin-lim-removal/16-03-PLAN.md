**Data pipeline (inside train or as helper):**
- Load audio files from `audio_dir` (use torchaudio.load, support WAV/FLAC/MP3)
- Load the saved model from `model_path` to get its `AudioSpectrogram` for mel computation
- Create a PyTorch Dataset that:
  1. Loads audio files and normalizes to [-1, 1]
  2. Cuts random segments of `config.segment_size` samples
  3. Computes mel via `AudioSpectrogram.waveform_to_mel()` (VAE's log1p format)
  4. Returns `(mel_segment, waveform_segment)` pairs
- DataLoader with `batch_size=config.batch_size`, `shuffle=True`, `drop_last=True`

**Data augmentation (TRAIN-03):**
- Apply augmentation to discriminator inputs (not generator inputs) to prevent overfitting:
  - Random gain variation (+/- 3dB)
  - Random noise injection (SNR 30-50dB)
  - These are simple and effective for small datasets
- Do NOT use the existing AugmentationPipeline (it modifies waveforms before mel computation, which is not what we want -- we want to augment the discriminator's real/fake audio inputs)

**Training loop (GAN alternation per research Pattern 1):**
- Create generator, MPD, MSD from config
- Create AdamW optimizers: optim_g (lr=config.learning_rate), optim_d (lr=config.learning_rate * 0.5 -- lower for discriminator overfitting prevention)
- Create ExponentialLR schedulers (gamma=config.lr_decay)
- If checkpoint provided: restore all state dicts (generator, mpd, msd, optim_g, optim_d, sched_g, sched_d, start_epoch)
- For each epoch:
  - For each batch:
    - Discriminator step: generate fake audio (detached), compute disc losses on augmented real+fake, backprop discriminator
    - Generator step: generate fake audio, compute gen loss + mel loss (weight 45) + feature matching loss, backprop generator
  - Step schedulers
  - Emit VocoderEpochMetrics via callback (gen_loss, disc_loss, mel_loss, feature_loss, lr, eta)
  - Every N epochs (e.g., every 10 or 20): generate a preview audio sample and emit via callback
  - Check cancel_event -- if set, save checkpoint immediately and return

**Cancel/checkpoint save (TRAIN-05):**
- On cancel_event.is_set(): build vocoder_state dict with checkpoint sub-dict containing all training state (generator, mpd, msd, optimizers, schedulers, current epoch)
- Save into the .distillgan model file by:
  1. Load the existing model file with torch.load
  2. Set save_dict["vocoder_state"] = vocoder_state_with_checkpoint
  3. torch.save back to same path
- Return the vocoder_state dict

**Completion:**
- On normal completion: build final vocoder_state WITHOUT checkpoint sub-dict (only generator state_dict, config, training_metadata)
- Save into .distillgan file same as cancel path but without checkpoint key
- Return vocoder_state dict

```python
class HiFiGANVocoder(VocoderBase):
    def __init__(self, vocoder_state: dict, device: str = "auto"):
        # Extract config and generator weights from vocoder_state
        config = HiFiGANConfig(**vocoder_state["config"])
        self._generator = HiFiGANGenerator(config)
        self._generator.load_state_dict(vocoder_state["generator_state_dict"])
        self._generator.eval()
        self._generator.remove_weight_norm()  # Fuse for inference speed
        self._sample_rate = config.sample_rate
        # Resolve device
        from distill.hardware.device import select_device
        self._device = select_device(device)
        self._generator.to(self._device)

    def mel_to_waveform(self, mel: torch.Tensor) -> torch.Tensor:
        # Input: [B, 1, 128, T] (VAE format, log1p normalized)
        # 1. Squeeze channel dim: [B, 128, T]
        # 2. Undo log1p: expm1(clamp(mel, min=0)) -> linear power mel
        # 3. Forward through generator: [B, 1, T*512]
        # Note: NO MelAdapter needed -- trained on VAE mels directly
        with torch.inference_mode():
            wav = self._generator(mel_input)
        return wav  # [B, 1, samples]

    @property
    def sample_rate(self) -> int:
        return self._sample_rate  # 48000 (trained on VAE's native rate)

    def to(self, device) -> HiFiGANVocoder:
        self._generator.to(device)
        self._device = device
        return self
```
