Rewrite MelAdapter.convert() to use a direct mel-domain conversion instead of the Griffin-Lim waveform round-trip. The approach:

**Direct filterbank transfer matrix approach:**
1. Build the HTK mel filterbank (project's format: 128 bands, 48kHz, n_fft=2048) using `torchaudio.functional.melscale_fbanks`
2. Build the Slaney mel filterbank (BigVGAN's format: 128 bands, 44.1kHz, n_fft=2048) using `librosa.filters.mel` (Slaney normalization)
3. Compute a pseudo-inverse transfer: `transfer = slaney_fb @ pinverse(htk_fb)` to map from HTK mel space to linear STFT space to Slaney mel space
4. Apply: `mel_bigvgan = log(clamp(transfer @ expm1(mel_vae), 1e-5))`

This avoids the waveform round-trip entirely. The mel-domain transfer is:
- **Faster** (matrix multiply vs Griffin-Lim iterations)
- **No quality loss** from Griffin-Lim approximation
- **Deterministic** (no random phase reconstruction)
